I'm going to test some cool things I put in the style.css file here.

List of findings:

  • The latest version of jQuery that is supported is 1.12.4, which is unfortunate.
  • Both bold and italics are broken on the DSi
  • There is only one font: the default font
  • em as a CSS width is broken
  • How the DSi deals with font sizes is cursed.
    • All sizes 1px to 9px are the same with the same element height;
    • After that, the element's height starts to linearly increase, but the font itself does not increase in size until 12px.
    • Sizes 12px to 14px are the same, and sizes 15px and up are the same.
    • You can test it below using a DSi.
  • The DSi doesn't support CORS
  • Array.prototype.forEach doesn't exist, as opposed to standard Opera 9.5
  • HTML5 canvases are scuffed on the DSi
    • It can only draw about 10 fps;
    • It is very limited, with no support for drawing font, only basic shapes and lines.
  • The static framerate for the DSi browser is about 10 fps, but it increases to about 20 fps if the screen is wiggled.
    My guess here is that forcing the DSi to render new parts of the webpage increases the refresh rate.
